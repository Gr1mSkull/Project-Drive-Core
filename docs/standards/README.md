## Rules

- Standards define precise fields, formats, compatibility rules, and evidence requirements.
- ADRs explain **why** a standard exists; standards define **how** identity shall be recorded.
- Do not duplicate complete normative rules in supporting documents — cross-reference the standard.
- An **Approved** standard is normative engineering documentation; it does not by itself mean runtime mechanisms are implemented or verified.
- Canonical ADR references use `ADR-NNN` (e.g. ADR-015). Originating request aliases are not second ADR IDs.
