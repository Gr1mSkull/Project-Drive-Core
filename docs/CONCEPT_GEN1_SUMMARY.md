# DriveCore Gen1 — Итоговая концепция (E30)

Сжатый перенос диалога «Разработка PDM для E30». Использовать как базовое ТЗ при возврате к проекту.

## 1. Общая идея

Центральный **DriveCore Controller (DCC)** объединяет PDM, VCM, диагностику, CAN gateway и базовый data logging.

**Цель:** упростить проводку гоночного E30, повысить надёжность и диагностируемость.

## 2. Узлы системы

| Узел | Роль |
|------|------|
| DCC | PDM + VCM + диагностика + gateway + логи |
| ECU | Только двигатель |
| Button Box | Умная клавиатура, события по CAN |
| Планшет | Web UI (клиент), не CAN-узел |

## 3. Нагрузки E30

EHPS, электропомпа, 2× вентиляторы, бензонасос, ECU, свет, салон, стеклоподъёмники, резерв.

## 4. Каналы

- 2 × 60 A, 4 × 30 A, 8 × 15 A, 8 × 5 A, 2 × H-bridge
- 20% запас каналов
- Каналы не привязаны жёстко к функции — настраиваются конфигом

## 5. Режимы автомобиля

OFF → MASTER ON → IGNITION → PRIME → READY → ENGINE RUN → RACE → COOL DOWN

## 6. Button Box

STM32 + CAN FD. События: press, release, long press, hold, rotate. Не управляет нагрузками.

## 7. Масса

Внешний физический контактор (регламент). DCC только мониторит.

## 8. Разъёмы

Deutsch DTM (сигналы/CAN), DT (средние нагрузки), DTP (силовые).

## 9. Проводка

Один силовой хаб, без реле, минимум жгутов и соединений, ремонт в паддоке за минуты.

## 10. Принцип Gen1

> Hardware is fixed, behavior is fully configurable.

## 11. Позиция

DriveCore Gen1 = **центральный автомобильный контроллер**, а не PDM. PDM — лишь часть системы.
